# Product Launch Checklist

## Pre-Launch: 8 Weeks Before

### Market Research
- [ ] Define target audience and ICP (Ideal Customer Profile)
- [ ] Complete competitive analysis (use `/competitor-intel analyze`)
- [ ] Validate pricing strategy (use `/pricing-analyzer audit`)
- [ ] Identify key differentiators and positioning
- [ ] Map customer journey from awareness to activation
- [ ] Research target communities and channels

### Product Readiness
- [ ] Core features complete and tested
- [ ] Onboarding flow implemented and tested (use `/onboarding-optimizer audit`)
- [ ] Analytics and event tracking set up
- [ ] Error monitoring and alerting configured
- [ ] Performance testing completed
- [ ] Security audit passed

### Brand and Messaging
- [ ] Finalize brand positioning statement
- [ ] Write core messaging (headline, tagline, elevator pitch)
- [ ] Create messaging matrix for different personas
- [ ] Design visual brand assets (logo, colors, typography)
- [ ] Write boilerplate and press descriptions

---

## Pre-Launch: 4 Weeks Before

### Landing Page
- [ ] Build landing page (use `/saas-landing-builder create`)
- [ ] Audit landing page for conversions (use `/landing-page-cro audit`)
- [ ] Set up conversion tracking (signups, CTAs, forms)
- [ ] A/B test headline variations
- [ ] Mobile responsiveness verified
- [ ] Page speed optimized (<3s load time)
- [ ] SEO basics: title, meta description, OG tags
- [ ] AI search optimization (use `/geo-seo-auditor audit`)

### Content Preparation
- [ ] Write launch blog post
- [ ] Create product demo video (2-3 minutes)
- [ ] Design social media graphics (all platforms)
- [ ] Write email sequences (waitlist, launch day, follow-up)
- [ ] Prepare FAQ document
- [ ] Create help documentation / knowledge base

### PR and Outreach
- [ ] Build media contact list (journalists, bloggers, influencers)
- [ ] Write and distribute press release
- [ ] Prepare press kit (logos, screenshots, founder bio, fact sheet)
- [ ] Schedule influencer outreach
- [ ] Prepare ProductHunt submission (hunter, tagline, images)
- [ ] Write Hacker News "Show HN" post

### Early Access
- [ ] Launch waitlist or early access signup
- [ ] Set up referral mechanism for waitlist
- [ ] Begin beta program with select users
- [ ] Collect testimonials from beta users
- [ ] Create case studies from early adopters

### App Store (if applicable)
- [ ] Optimize app store listing (use `/aso-optimizer analyze`)
- [ ] Prepare screenshots and preview video
- [ ] Write app description with keywords
- [ ] Submit for app review (allow 1-2 weeks buffer)

---

## Pre-Launch: 1 Week Before

### Final QA
- [ ] Full end-to-end testing of critical paths
- [ ] Test signup and onboarding flow on all devices
- [ ] Test payment processing (all methods)
- [ ] Test email delivery (transactional and marketing)
- [ ] Load testing for expected traffic surge
- [ ] Verify all links and CTAs work

### Infrastructure
- [ ] Server scaling plan ready for traffic spike
- [ ] CDN configured and tested
- [ ] Database backup verified
- [ ] Monitoring dashboards set up
- [ ] On-call rotation scheduled for launch week
- [ ] Rollback plan documented

### Team Preparation
- [ ] Brief entire team on launch plan and timeline
- [ ] Assign roles: social media, support, engineering, PR
- [ ] Create internal communication channel for launch day
- [ ] Prepare customer support scripts and FAQs
- [ ] Schedule all social media posts
- [ ] Test all automated workflows (welcome emails, notifications)

### Final Content
- [ ] All blog posts scheduled
- [ ] Email sequences queued and tested
- [ ] Social media posts scheduled across platforms
- [ ] Community posts drafted (Reddit, forums, etc.)
- [ ] Partner communications sent

---

## Launch Day

### Morning (Pre-Launch)
- [ ] Final system health check
- [ ] Verify all monitoring is active
- [ ] Confirm team availability and roles
- [ ] Pre-load support queue with FAQ responses

### Launch Execution
- [ ] Remove beta/coming-soon flags
- [ ] Send launch email to waitlist
- [ ] Publish blog post
- [ ] Post on social media (all platforms simultaneously)
- [ ] Submit to ProductHunt
- [ ] Post on Hacker News (Show HN)
- [ ] Submit to Reddit (relevant subreddits, following rules)
- [ ] Notify press contacts
- [ ] Activate paid campaigns (if planned)

### Real-Time Monitoring
- [ ] Monitor server performance and uptime
- [ ] Watch conversion funnel in real-time
- [ ] Track social media mentions and engagement
- [ ] Respond to comments and questions within 1 hour
- [ ] Monitor support tickets and common issues
- [ ] Track press coverage and media mentions

### End of Day
- [ ] Compile Day 1 metrics snapshot
- [ ] Note any issues for immediate fixes
- [ ] Thank early users and supporters publicly
- [ ] Share internal update with team
- [ ] Plan Day 2 follow-up activities

---

## Post-Launch: Week 1

### Metrics Review
- [ ] Compile Week 1 metrics (use `/subscription-metrics calculate`)
- [ ] Analyze signup funnel conversion rates
- [ ] Review activation and onboarding completion
- [ ] Check churn/cancellation rate
- [ ] Assess traffic sources and channel performance
- [ ] Calculate initial CAC by channel

### User Feedback
- [ ] Review all user feedback and categorize
- [ ] Analyze support tickets for common themes (use `/review-sentiment analyze`)
- [ ] Identify top 3 user complaints and address
- [ ] Collect testimonials from satisfied users
- [ ] Conduct quick interviews with 5-10 new users

### Quick Fixes
- [ ] Fix critical bugs reported by users
- [ ] Address UX friction points in onboarding
- [ ] Optimize slow-performing pages
- [ ] Update FAQ based on common questions
- [ ] Improve error messages and edge cases

### Continued Outreach
- [ ] Follow up with journalists who didn't cover launch
- [ ] Respond to all ProductHunt/HN comments
- [ ] Engage in Reddit and community discussions
- [ ] Send personal thank-you notes to key supporters
- [ ] Share early traction numbers publicly (if positive)

---

## Post-Launch: Month 1

### Optimization Cycle
- [ ] Full funnel analysis (use `/plg-funnel-analyzer audit`)
- [ ] Landing page optimization based on data (use `/landing-page-cro audit`)
- [ ] Email sequence optimization (open rates, click rates)
- [ ] Onboarding flow iteration based on drop-off data
- [ ] Pricing validation with early customer data

### Content Marketing Engine
- [ ] Publish 4+ blog posts (weekly cadence)
- [ ] Create 1 comprehensive guide or whitepaper
- [ ] Start SEO content calendar
- [ ] Begin building email newsletter
- [ ] Guest posting outreach (3-5 targets)

### Paid Experiments
- [ ] Test 2-3 paid channels with small budgets ($500-1000 each)
- [ ] Set up proper attribution tracking
- [ ] Define CAC targets per channel
- [ ] Kill underperforming channels quickly
- [ ] Scale channels that meet CAC targets

### Retention Focus
- [ ] Implement NPS or satisfaction survey
- [ ] Set up churn risk monitoring (use `/retention-playbook signals`)
- [ ] Create re-engagement email sequence
- [ ] Build customer success touchpoint calendar
- [ ] Document ideal customer journey and activation milestones

### Growth Foundation
- [ ] Set up referral program
- [ ] Build integration with 1-2 popular tools
- [ ] Create API documentation (if B2B)
- [ ] Plan next feature release based on user requests
- [ ] Establish monthly metrics review cadence
