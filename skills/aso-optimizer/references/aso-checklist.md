# ASO Checklist

Comprehensive checklists for App Store Optimization before and after launch.

## Pre-Launch ASO Checklist

Complete these items before submitting your app to the store.

### Keyword Research
- [ ] Identify 20-30 target keywords relevant to your app
- [ ] Categorize keywords by intent (navigational, informational, transactional)
- [ ] Analyze competitor keyword usage in titles, subtitles, and descriptions
- [ ] Prioritize long-tail keywords for new apps with low authority
- [ ] Validate that target keywords match your app's core functionality

### Title and Subtitle
- [ ] Include the highest-value keyword in the app title
- [ ] Keep the title under 30 characters (both platforms)
- [ ] Write an iOS subtitle using secondary keywords (30 chars max)
- [ ] Write a Google Play short description with a value proposition (80 chars max)
- [ ] Ensure brand name is recognizable but does not consume all title space

### Description
- [ ] Write a compelling first 3 lines (visible before "Read More")
- [ ] Structure the full description with line breaks and bullet points
- [ ] Include target keywords naturally throughout the text
- [ ] Add a clear call-to-action near the end
- [ ] Keep description between 2,000 and 4,000 characters for optimal indexing

### Keywords Field (iOS Only)
- [ ] Fill all 100 characters in the keyword field
- [ ] Use commas without spaces to separate terms
- [ ] Avoid repeating words from the title or subtitle
- [ ] Include common misspellings and synonyms
- [ ] Use singular forms (Apple indexes both singular and plural)

### Visual Assets
- [ ] Prepare 6-10 screenshots showing key features
- [ ] Design the first 2 screenshots to communicate core value instantly
- [ ] Add captions and callout text to screenshots
- [ ] Create an app preview video (15-30 seconds recommended)
- [ ] Ensure the app icon is clean, recognizable, and stands out at small sizes

### Category and Metadata
- [ ] Select the most relevant primary category
- [ ] Choose a secondary category for additional browse visibility
- [ ] Set the appropriate age rating
- [ ] Fill in the privacy policy URL
- [ ] Complete all required metadata fields for both platforms

### Localization
- [ ] Identify top 5 target markets by potential user base
- [ ] Translate and localize metadata for each target locale
- [ ] Localize screenshots and preview videos
- [ ] Research locale-specific keywords (direct translations often miss intent)

## Post-Launch Optimization Cycle

ASO is not a one-time task. Follow this recurring cycle.

### Weekly Tasks
- Monitor keyword ranking changes for your top 10 keywords
- Track daily download trends and conversion rate
- Read and respond to new user reviews (especially negative ones)
- Check competitor listing changes for new keyword opportunities

### Bi-Weekly Tasks
- Analyze conversion rate by traffic source (search, browse, referral)
- Review screenshot and video engagement metrics if available
- Adjust keyword field based on ranking movement data
- Test new promotional text variations (iOS - no review needed)

### Monthly Tasks
- Conduct a full keyword audit and refresh underperforming terms
- Update screenshots if new features have launched
- Benchmark against top 5 competitors on metadata quality
- Review and update the "What's New" section with each release
- Analyze rating trends and identify recurring complaints

### Quarterly Tasks
- Perform a complete metadata overhaul if conversion rates have plateaued
- Re-evaluate primary and secondary category selection
- Refresh localization for top markets
- Audit visual assets against current design trends
- Review ASO health score and set targets for next quarter

## A/B Test Strategies

Systematic testing improves conversion rates over time.

### What to Test
- **App icon**: Color schemes, illustration style, symbol prominence
- **Screenshots**: Order, caption text, background color, feature emphasis
- **Title/Subtitle**: Keyword order, brand placement, value proposition phrasing
- **Short description** (Google Play): Different benefit angles, CTA variations
- **Preview video**: With video vs without, different opening frames

### Testing Best Practices
- Test one variable at a time to isolate impact
- Run tests for at least 7 days to account for weekly traffic patterns
- Require statistical significance (95% confidence) before declaring a winner
- Use Google Play's built-in Store Listing Experiments for Android
- For iOS, use Apple's Product Page Optimization feature
- Document every test with hypothesis, variant details, and results
- Re-test winning variants against new challengers periodically

### Common Test Pitfalls
- Ending tests too early due to impatience
- Testing during seasonal spikes or marketing campaigns (skewed data)
- Changing multiple elements simultaneously
- Ignoring segment differences (organic vs paid traffic may react differently)

## Seasonal Optimization Tips

Align your ASO strategy with seasonal demand patterns.

### Major Seasonal Windows
- **New Year (January)**: Fitness, productivity, finance, and self-improvement apps surge
- **Back to School (August-September)**: Education, note-taking, and organization apps peak
- **Holiday Season (November-December)**: Gaming, shopping, entertainment, and gift-related apps spike
- **Summer (June-July)**: Travel, fitness, and entertainment apps see increased demand
- **Tax Season (February-April)**: Finance, accounting, and tax preparation apps gain traction

### Seasonal Strategy
- Update keywords 2-3 weeks before a seasonal window begins
- Refresh screenshots and descriptions to highlight seasonal relevance
- Prepare seasonal promotional text (iOS) for quick, review-free updates
- Create seasonal app preview videos if relevant
- Plan feature releases to coincide with peak demand periods
- Revert seasonal changes once the window passes to avoid irrelevant messaging

## Review Solicitation Best Practices

Ratings and reviews directly impact ASO ranking and conversion.

### When to Ask for Reviews
- After a positive user action (completing a task, achieving a goal)
- When the user has been active for at least 3-5 sessions
- Never after a crash, error, or frustrating experience
- Avoid asking during onboarding or first session
- Space requests at least 60 days apart for the same user

### How to Ask
- Use the native `SKStoreReviewController` (iOS) or Google Play In-App Review API
- These native prompts have higher completion rates than custom dialogs
- Apple limits the native prompt to 3 times per 365-day period per user
- Pre-qualify with a "sentiment gate" - ask "Are you enjoying the app?" first
- If the user says no, route them to a feedback form instead of the store

### Managing Negative Reviews
- Respond to every 1-star and 2-star review within 24-48 hours
- Be professional, empathetic, and offer a concrete resolution path
- Ask the user to update their review after the issue is resolved
- Track recurring complaints and feed them into the product roadmap
- Never offer incentives in exchange for positive reviews (violates store policies)

### Rating Goals
- Target a minimum 4.0 average rating for credibility
- Aim for 4.5+ to appear in "top rated" browse sections
- Focus on total review count - higher counts reduce the impact of occasional low ratings
- Monitor rating trends weekly to catch emerging issues early
